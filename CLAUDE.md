## Overview

Bun 製 TypeScript による型補完で快適に書けるタスクランナー。

## Reference

- 設計情報: `@docs/design.md`

## 作業ルール

- コードは常にメンテナンス製とテスト容易性を最も重要視する
- 時間の節約・開発スループットよりも、確実とトレーサビリティを優先する
- 後方互換性や例外処理のために処理を複雑にせず、シンプルで唯一の正解のために `DRY` な実装を行う

## Folder Rule

- `@src/ `- Bunのコード本体
- `@test/` - テストコードを格納する

## Definition of Done

- `scripts/sanity.sh` を実行し、型エラー・フォーマット・全件テストが全てパスすること
